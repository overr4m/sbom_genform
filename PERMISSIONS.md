# Required Filesystem Permissions

This document lists every path the **secsbom-pipeline** CLI reads from or writes to,
and the minimum access level required at each.

---

## Legend

| Level | Meaning |
|-------|---------|
| **R** | Read-only |
| **RW** | Read + Write (files created/overwritten) |
| **RWX** | Read + Write + Execute (directory traversal or binary execution) |
| **X** | Execute only (binary invocation) |

---

## Input paths

| Path | Level | Description |
|------|-------|-------------|
| `<project_dir>/` | **R** | Source project to analyse (env `PROJECT_DIR`, default `examples/project_inject`). The tool only reads manifests and source files; it never modifies the project. |
| `.env` | **R** | Optional dotenv file loaded at startup. |

---

## Output paths

All output directories are created automatically if they do not exist.

| Path | Level | Description |
|------|-------|-------------|
| `<output_dir>/` | **RW** | Root output directory (env `OUTPUT_DIR`, default `secgensbom_out`). |
| `<output_dir>/app-bom-cdxgen.json` | **RW** | Raw SBOM generated by cdxgen / cyclonedx-py. |
| `<output_dir>/app-bom-dedup.json` | **RW** | Deduplicated SBOM. |
| `<output_dir>/merged-bom-signed.json` | **RW** | Final SBOM with embedded SHA-256 signature. |
| `<output_dir>/merged-bom-signed.sig` | **RW** | Detached SHA-256 signature file. |
| `<output_dir>/vulns-normalized.json` | **RW** | Merged vulnerability report (all scanners). |
| `<output_dir>/trivy/` | **RW** | Trivy scan results. |
| `<output_dir>/clair/` | **RW** | Clair scan results. |
| `<output_dir>/dependency-check/` | **RW** | OWASP Dependency-Check reports (all formats). |
| `<reports_dir>/excel/` | **RW** | Excel (.xlsx) report + `.sig` file (env `REPORTS_DIR`, default `secgensbom_reports`). |
| `<reports_dir>/docx/` | **RW** | Word (.docx) report + `.sig` file. |
| `<reports_dir>/odt/` | **RW** | ODT report + `.sig` file. |
| `sbom_pipeline.log` | **RW** | Log file written in the working directory. |

---

## Dependency-Check NVD cache

| Path | Level | Description |
|------|-------|-------------|
| `.dependency-check-data/` | **RW** | NVD vulnerability database cache (env `DEP_CHECK_DATA`). Shared across runs to avoid re-downloading. |

---

## Temporary paths (Docker / DinD mode)

These paths are relevant only when the pipeline spawns child containers
(Dependency-Check, clairctl) via the Docker socket.

| Path | Level | Description |
|------|-------|-------------|
| `/var/run/docker.sock` | **RW** | Docker daemon socket. Required to launch scanner containers. |
| `<HOST_PROJECT_DIR>` | **R** | Host-side project path bind-mounted into child containers (env `HOST_PROJECT_DIR`). |
| `<HOST_OUTPUT_DIR>` | **RW** | Host-side output path bind-mounted into child containers (env `HOST_OUTPUT_DIR`). |

---

## Executables

These binaries must be on `PATH` and executable by the process user.

| Binary | Level | Description |
|--------|-------|-------------|
| `trivy` | **X** | Trivy vulnerability scanner. |
| `npx` / `node` | **X** | Node.js runtime used to invoke cdxgen for non-Python projects. |
| `cyclonedx-py` | **X** | CycloneDX generator for Python projects (optional; falls back to cdxgen). |
| `docker` (CLI) | **X** | Docker client used to spawn Dependency-Check and clairctl containers. |

---

## npm / npx cache (cdxgen)

| Path | Level | Description |
|------|-------|-------------|
| `$NPM_CONFIG_CACHE` (default `~/.npm`) | **RW** | npm package cache used by `npx @cyclonedx/cdxgen`. The process user's home directory must exist and be writable. |

---

## Notes

- All paths above can be overridden via environment variables (see `src/sbom_pipeline/config.py`).
- When running inside Docker, volume mounts must reflect the same permission requirements (see `docker-compose.yml`).
- The process runs as an unprivileged user (`uid 1001`, group `sbom`). Ensure bind-mounted host directories are accessible to that uid, e.g. `chown -R 1001:1001 secgensbom_out secgensbom_reports`.
